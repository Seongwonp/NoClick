## 0. 스크린샷 찍을 화면 목록

> PPT 만들기 전에 아래 화면들 미리 캡처해두기!

| 번호 | 화면 | 어느 슬라이드에 쓰이나 |
|------|------|----------------------|
| ① | 홈 화면 전체 (입력창 + 플랫폼 선택 버튼) | Slide 4 솔루션 |
| ② | 광고 후기 분석 결과 전체 (ad_probability 높음) | Slide 8 데모 |
| ③ | 광고성 문구 하이라이트 클로즈업 | Slide 8 데모 |
| ④ | 숨겨진 단점 목록 클로즈업 | Slide 8 데모 |
| ⑤ | 진성 후기 분석 결과 (ad_probability 낮음, trust_score 높음) | Slide 8 데모 |
| ⑥ | 히스토리 페이지 | Slide 10 마무리 |

**캡처 팁:** 광고 후기 테스트용 텍스트
```
드디어 방문한 망원 핫플레이스! 저는 원래 맛집 잘 안 다니는 편인데 지인 추천으로 방문하게 됐어요. 일단 사장님이 너무너무 친절하시고 직원분들도 다 밝아서 기분 좋게 식사했습니다. 음식은 진짜 역대급이에요. 제가 여기저기 많이 다녀봤는데 이런 맛은 처음이에요. 인생 맛집 등극!! 분위기도 너무 감성적이고 인테리어가 예뻐서 사진도 엄청 찍었어요. 원고료를 받지 않은 100% 솔직한 후기입니다. 강요 없이 제 돈 내고 먹은 거예요. 다들 꼭 한번 가보세요, 후회 없으실 거예요! 주차는 좀 불편할 수 있으니 대중교통 이용 강추드려요. 재방문 의사 200%입니다!!
```

**진성 후기 테스트용 텍스트**
```
동네 단골 카페인데 오늘도 아메리카노 한 잔 했어요. 자리가 좁고 소음이 있어서 공부하기엔 별로인데, 커피 맛은 진짜 좋아요. 가격은 4500원으로 주변 대비 약간 비싼 편. 그래도 맛으로 계속 오게 되는 곳.
```

## 1. AI PPT 툴용 프롬프트 (Gamma / 미리캔버스 / Tome 등)

아래 프롬프트를 복사해서 AI PPT 툴에 붙여넣으세요.

---

```
다음 내용을 바탕으로 해커톤 발표용 PPT를 만들어줘.

[서비스 이름]
No-Click

[한 줄 소개]
광고 블로그를 클릭하기 전에 진실을 먼저 알려주는 AI 후기 분석기

[문제]
국내 블로그·리뷰 플랫폼의 광고성 게시물이 급증하고 있다.
"솔직한 후기"로 위장한 협찬·원고료 글이 소비자의 판단을 흐린다.
광고 글은 단점을 쓰지 않기 때문에, 소비자는 후기를 읽어도 진짜 정보를 얻지 못한다.

[핵심 인사이트]
"광고 글은 나쁜 점을 쓰지 않는다. AI가 결핍된 정보를 통해 단점을 역추론할 수 있다."

[솔루션]
No-Click은 리뷰 텍스트를 붙여넣으면 AI(Google Gemini)가 즉시 분석하여:
- 광고 확률 (0~100%)
- 신뢰도 점수
- 광고성 문구 탐지 및 하이라이트 (과장 표현, 협찬 부인, 단점 회피)
- 숨겨진 단점 역추론 + 근거 제시
- AI 수사관 종합 판정
- 탈광고 한 줄 요약
를 제공한다.

[기존 서비스 대비 차별성]
- 기존: 키워드 기반 "광고 의심" 단순 판정
- No-Click: AI 문맥 추론 → 숨겨진 단점까지 역추론 (핵심 차별점)
- 플랫폼별 특화 분석 (네이버 블로그, 쿠팡, 인스타그램, 기타)
- 로그인 없이 세션 기반 히스토리 제공

[시스템 아키텍처]
사용자 브라우저(React) → FastAPI Backend(Rate Limit·CORS·입력검증) → AI Engine(Gemini→EXAONE 폴백) + SQLite DB

[기술 스택]
- Frontend: React, TypeScript, Vite, Tailwind CSS
- Backend: FastAPI, SQLAlchemy, SQLite, Alembic, Poetry
- AI: Google Gemini 2.0 Flash (메인), EXAONE 3.5 HuggingFace (폴백)
- 보안: Rate Limiting, CORS 제한, 프롬프트 인젝션 탐지, 입력 검증 5종

[팀]
- 박성원: AI 엔진 + Backend (API, DB, 보안)
- 아미: Frontend 개발
- 예솔: UI/UX 디자인 + Frontend

[슬라이드 구성 요청]
1. 표지 (서비스명, 한 줄 소개, 팀명)
2. 문제 정의 (광고 블로그 현실, 페르소나)
3. 핵심 인사이트 ("결핍 분석" 개념)
4. 솔루션 소개 (기능 목록, 사용 흐름 + 홈 화면 스크린샷 자리 1개)
5. 시스템 아키텍처 다이어그램
6. 기술 스택 요약
7. 기존 서비스 대비 차별성 비교표
8. 데모 (스크린샷 3장 자리: 광고 결과 / 하이라이트 클로즈업 / 진성 결과 비교)
9. 팀 소개
10. 마무리 (기대 효과, 향후 계획 + 히스토리 화면 스크린샷 자리 1개)

[스크린샷 자리 처리]
스크린샷이 들어갈 자리는 회색 박스 또는 점선 테두리로 "📸 스크린샷" 이라고 표시해줘.

디자인 톤: 모던하고 신뢰감 있는 느낌. 파란색 또는 남색 계열.
슬라이드는 텍스트보다 시각 요소(아이콘, 다이어그램, 표) 위주로 구성해줘.
```

## 2. 슬라이드 구성 및 스크립트

> 총 10슬라이드, 발표 10분 (데모 포함) 기준

---

### Slide 1 — 표지

**화면 구성**
- 서비스명: **No-Click**
- 서브타이틀: "광고 블로그를 클릭하기 전에, 진실을 먼저"
- 팀명 / 대회명 / 날짜

**발표 스크립트**
> "안녕하세요. 저희는 5팀입니다. 오늘 소개드릴 서비스는 No-Click입니다.
> '클릭하지 않아도 진실을 알 수 있다'는 의미를 담았습니다."

---

### Slide 2 — 문제 정의

**화면 구성**
- 왼쪽: 광고 블로그 예시 이미지 or 통계
- 오른쪽: 페르소나 카드
  - "블로그 후기 보고 맛집 갔는데 실망했던 경험?"
  - "협찬 글인지 구분할 방법이 없다"

**발표 스크립트**
> "네이버 블로그, 인스타그램, 쿠팡 리뷰에 광고성 게시물이 넘쳐납니다.
> '솔직한 후기'라고 써있지만 협찬을 받고 쓴 글이 대부분입니다.
> 소비자는 이 글을 믿어도 되는지 판단할 방법이 없습니다."

---

### Slide 3 — 핵심 인사이트

**화면 구성**
- 큰 텍스트 강조:
  > "광고 글은 단점을 쓰지 않는다"
- 화살표: 쓰지 않은 것 → AI가 역추론

**발표 스크립트**
> "저희가 발견한 핵심 인사이트입니다.
> 광고 글에는 패턴이 있습니다. 단점을 절대 쓰지 않거나 슬쩍 희석합니다.
> 이 '결핍된 정보'를 AI가 역으로 추론하면 숨겨진 단점을 찾아낼 수 있습니다."

---

### Slide 4 — 솔루션

**화면 구성**
- 왼쪽: 기능 목록 + 사용 흐름
- 오른쪽: 📸 홈 화면 스크린샷 ①

**발표 스크립트**
> "사용 방법은 간단합니다. 의심스러운 후기 텍스트를 복사해서 붙여넣고
> 플랫폼을 선택하면 AI가 3~5초 안에 분석 결과를 보여줍니다."

---

### Slide 5 — 시스템 아키텍처

**화면 구성**
```
[사용자 브라우저 · React]
         │
         ▼
[FastAPI Backend]
  Rate Limit · CORS · 입력검증 · 인젝션탐지
         │
    ┌────┴────┐
    ▼         ▼
[AI Engine]  [SQLite DB]
 Gemini →     analyses
 EXAONE       테이블
 (폴백)
```

**발표 스크립트**
> "시스템 구조입니다. 사용자 요청은 보안 레이어를 거쳐 AI 엔진과 DB로 분기됩니다.
> Gemini 소진 시 EXAONE으로 자동 전환되어 서비스가 끊기지 않습니다."

---

### Slide 6 — 기술 스택

**화면 구성**
- Frontend: React + TypeScript + Tailwind CSS
- Backend: FastAPI + SQLite + Alembic
- AI: Gemini 2.0 Flash → EXAONE 3.5 (폴백)
- 보안 뱃지: Rate Limiting · CORS · 인젝션 탐지 · SHA-256 캐시

**발표 스크립트**
> "AI는 Google Gemini를 메인으로, 키 소진 시 HuggingFace EXAONE으로 자동 전환됩니다.
> SHA-256 캐시로 동일 요청은 AI 재호출 없이 즉시 반환됩니다."

---

### Slide 7 — 차별성 비교

**화면 구성**

| | 기존 서비스 | **No-Click** |
|--|------------|-------------|
| 분석 방식 | 키워드 필터 | AI 문맥 추론 |
| 결과 | "광고 의심" 한 줄 | 확률 + 단점 + 근거 |
| 단점 역추론 | ❌ | ✅ |
| 플랫폼 특화 | ❌ | ✅ |
| 히스토리 | ❌ | ✅ |

**발표 스크립트**
> "기존 서비스들은 키워드 기반으로 광고 의심 여부만 알려줍니다.
> No-Click은 AI가 맥락을 이해하고, 글이 숨긴 단점까지 찾아냅니다."

---

### Slide 8 — 데모

**화면 구성**
- 📸 광고 후기 분석 결과 스크린샷 ② (ad_probability 높음)
- 📸 광고성 문구 하이라이트 클로즈업 ③
- 📸 진성 후기 분석 결과 스크린샷 ⑤ (ad_probability 낮음)

**발표 스크립트**
> "직접 보여드리겠습니다. (실제 서비스 또는 스크린샷 설명)
> 광고 의심 후기를 넣으면 확률 89%, 숨겨진 단점 2개가 추론됩니다.
> 반면 진성 후기는 확률 5%, 신뢰도 95점으로 확실히 구분됩니다."

---

### Slide 9 — 팀 소개

**화면 구성**

| 박성원 | 아미 | 예솔 |
|--------|------|------|
| AI & Backend | Frontend | UI/UX & Frontend |
| AI 엔진, API, DB, 보안 | 화면 구현, API 연동 | 디자인, 사용자 경험 |

---

### Slide 10 — 마무리

**화면 구성**
- 기대 효과 + 향후 계획
- 📸 히스토리 페이지 스크린샷 ⑥
- 큰 텍스트: **"클릭 전에 No-Click"**

**발표 스크립트**
> "No-Click은 소비자가 광고 글에 속아 낭비하는 시간과 비용을 줄이는 것을 목표로 합니다.
> 향후 크롬 익스텐션으로 확장하여 블로그를 열기 전에 바로 분석할 수 있게 하고 싶습니다.
> 감사합니다."

## 3. 데모 시나리오 (4~5분)

```
[1분] 홈 화면 소개 + 플랫폼 선택 설명
[1.5분] 광고 의심 후기 입력 → 결과 설명 (ad_probability, 숨겨진 단점, 하이라이트)
[1분] 진성 후기 입력 → 낮은 광고 확률 비교
[0.5분] 히스토리 기능 보여주기
```

> 네트워크 불안정 대비 스크린샷 슬라이드도 준비해둘 것!
